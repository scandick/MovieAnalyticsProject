# Краткая шпаргалка по Git

## Основные команды

```bash
git clone <url>             # Клонировать репозиторий 
cd <название папки>         # Перейти в сам репозиторий 
git status                  # Показать статус изменений
git add <file>              # Добавить файл в индекс
git add .                   # Добавить все изменённые файлы (лучше испольщовать его) 
git commit -m "Сообщение"   # Зафиксировать изменения 
git push origin <branch>    # Отправить изменения в удалённый репозиторий 
git pull                    # Получить изменения из удалённого репозитория
```


## Ветки

```bash
git branch                  # Показать список веток
git branch <branch-name>    # Создать новую ветку 
git checkout <branch-name>  # Переключиться на ветку 
git checkout -b <branch-name> # Или создать и переключиться на ветку
git merge <branch-name>     # Слить ветку в текущую
git branch -d <branch-name> # Удалить ветку
```

Сливать ветки можно в GitLab, когда появляется новая ветка, вверху репозитория появиться кнопка "Crteate merge request" 

## Просмотр истории

```bash
git log                     # Показать историю коммитов
git log --oneline           # Краткая история коммитов
git diff                    # Показать изменения в файлах
```

## Отмена изменений

```bash
git restore <file>          # Отменить изменения в файле
git reset HEAD <file>       # Убрать файл из индекса
git revert <commit-id>      # Создать коммит, отменяющий указанный
git reset --hard <commit-id> # Сбросить до указанного коммита
```

## Работа с удалённым репозиторием

```bash
git remote add origin <url> # Добавить удалённый репозиторий
git remote -v               # Показать удалённые репозитории
git push -u origin <branch> # Отправить ветку в удалённый репозиторий
```

## Полезные команды

```bash
git stash                   # Сохранить изменения во временное хранилище
git stash pop               # Применить и удалить последние сохранённые изменения
git fetch                   # Получить обновления из удалённого репозитория без слияния
```

# Стандартный сценарий использования
```bash
https://git-scm.com/install/windows # скачаем Git, установим
Win + R: cmd -> Enter # открываем консоль
git --version # проверим что гит установился

# создаем папку на компе, в которой будет удобно работать, копируем путь к ней <путь к папке>
# продолжаем в консоли:
cd '<путь к папке>' # теперь мы "сидим" в этой папке через консоль 
git clone https://github.com/scandick/MovieAnalyticsProject.git # клонируем этот репозиторий себе в папку и убеждаемся, что появилась структура папок, как здесь

git branch <придумайте имя вашей ветки>    # создать новую вою ветку (моя primakas, по ФИО)
git checkout <ваша ветка> # переключиться на вашу ветку

# далее можно добавлять документы, писать код, что угодно внутри папки проекта
# а когда внесли изменение и необходимо его зафиксировать делаем:
git pull origin develop # стягиваем себе в локальную папку состояние ветки develop (т.к. в него могли вносить изменения сокомандники)
git add . # добавляем все изменения
git commit -m 'добавлена func1() в parser.py'  # коммитим, т.е. объясняем языком что мы изменили/добавили
git push origin <ваша ветка> # отправляем изменения сюда, в github (т.е. в удаленный репозиторий)
# изменения в вашей ветке

# но, чтобы вести совместную разработку, нам необходимо добвалять все изменения в одну ветку - develop (уже создана)
git checkout develop # переключаемся на ветку develop
git merge <ваша ветка> # мерджим (объединяем) ветку develop с <вашей веткой>
git push origin develop # отправляем изменения уже в develop

UPD: Если повторить этот процесс несколько раз, то становится очень даже быстро и понятно!
```

## Советы

- Используйте `.gitignore` для исключения ненужных файлов.
- Регулярно делайте `git pull`, чтобы избежать конфликтов.
- Пишите понятные сообщения для коммитов.
- Проверяйте `git status` перед любыми действиями.
